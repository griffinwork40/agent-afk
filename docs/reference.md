# Reference

Full env-var table, slash-command taxonomy, plugin/marketplace deep-dive. For architecture and runtime concepts, see [`architecture.md`](architecture.md). For build/test/release, see [`development.md`](development.md).

## Environment variables

**Authoritative catalogue:** [`env-registry.md`](env-registry.md) (generated from `src/config/env.ts`).

The registry lists every env var the runtime reads — currently 111 entries across model / auth / telegram / paths / daemon / worktree / mcp / browser / debug / routing / process / misc categories. It's regenerated by `pnpm scan:env` and validated in CI (`pnpm scan:env:check`, `pnpm audit:env:check`) so the docs stay in sync with the code.

To add or change an env var: edit `src/config/env.ts` (add a getter on `env` + an entry in `ENV_REGISTRY`), run `pnpm scan:env`, and commit both the source change and the regenerated `docs/env-registry.{json,md}`. Direct `process.env.*` reads outside `src/config/env.ts` are blocked by `pnpm audit:env:check` in CI.

`.env.example` mirrors the most common ones for first-time setup. Quick reference for the absolute minimum to get going:

```env
# Required (Anthropic provider)
ANTHROPIC_API_KEY=sk-ant-api03-...

# Telegram bot (only if running the bot surface)
TELEGRAM_BOT_TOKEN=1234567890:ABC...
AFK_TELEGRAM_ALLOWED_CHAT_IDS=     # comma-separated chat IDs allowed to interact
```

Everything else (model selection, prompt caching, daemon, worktree management, debug toggles, color output, etc.) lives in the [registry](env-registry.md).

## CLI command reference

The `afk` CLI exposes two dozen top-level commands registered in `src/cli/index.ts`:

| Command | Purpose |
|---|---|
| `chat` | Single-turn message |
| `interactive` | REPL with full Agent SDK features (default when invoked without a subcommand) |
| `status` | Connection, API-key, model, bypass-mode status |
| `config` | Dump resolved configuration |
| `daemon` | Long-running headless agent (see `src/agent/daemon/`) |
| `login` | OAuth flow for `console.anthropic.com` |
| `plugin` | Manage `~/.afk/plugins/` (install / update / list / remove / enable / disable) |
| `marketplace` | Add / list / remove plugin marketplaces under `~/.afk/marketplaces/` |
| `doctor` | Environment self-check (Node version, API keys, paths, config) |
| `completion` | Print a shell completion script (`zsh`, `bash`, `fish`) |
| `telegram` | Manage the Telegram bot daemon (start, stop, status, logs, setup) |
| `farm` | Parallel multi-worktree speculative eval: spawn N isolated worktrees, run a subagent DAG, score branches, send a Telegram digest |
| `schedule` | Manage cron-scheduled headless tasks in `~/.afk/config/schedules.json` (add / list / remove / enable / disable / logs) |
| `improve` | Self-improvement pipeline over witness telemetry (scan / cards / propose / proposals / eval-*) |
| `queue` | Enqueue work for `afk daemon --trigger pull` (add / list / clear) |
| `worktree` | Manage AFK-created git worktrees (list / prune) |
| `migrate` | Import plugins / skills / MCP servers from Claude Code or Codex |
| `service` | Install/manage an always-on background process as a macOS LaunchAgent (install / uninstall / status / restart) |
| `update` | Self-update the `afk` CLI (`--check`, `--pin`) |
| `trace` | Inspect session witness traces (show / list / diff) |
| `bg` | Inspect persisted background-subagent job logs after the REPL exits (list / tail / replay) |
| `browser` | Connect / disconnect a real Chrome instance for browser automation |
| `provider` | Inspect / diagnose model-provider auth (auth / diagnose) |
| `shell-init` | Emit a shell wrapper enabling cd-on-exit for worktree sessions |

Aliases: `afk c` → `chat`, `afk i` → `interactive`, `afk s` → `status`.

All commands support `--format json` where machine-readable output makes sense (`status`, `config`, `doctor`, `plugin list`).

### Per-session overrides

```
--model <name>            # local | small | medium | large | opus | sonnet | haiku | fable | codex
--thinking <on|off|N>     # extended thinking budget
--max-output-tokens <n>
--temperature <n>
--max-turns <n>           # safety rail (default 100)
--max-budget-usd <n>
--task-budget <tokens>
--dangerously-skip-permissions  # force bypass mode (already the CLI default for new installs); disable with `afk config set permissionMode default`
--no-update-check         # suppress startup update check
--dump-prompt <path>      # write resolved system prompt to file (provenance included)
```

## REPL slash commands

The interactive REPL registers slash commands directly in TypeScript (`src/cli/slash/`) — they don't pass through to any external Claude Code subprocess.

### Core / session control
- `/help` — list all available slash commands (built-in + plugin-loaded)
- `/exit`, `/quit` — leave the REPL
- `/clear` — clear screen
- `/compact` — manually compact conversation history

### Information
- `/cost` — running cost for the session
- `/tokens` (alias `/ctx`) — SDK breakdown of context usage
- `/history` — print prior turns
- `/model` — show or switch active model
- `/tools` — list registered tools
- `/mcp` — show MCP server status ([full docs](mcp.md))
- `/limits` — show rate-limit / budget state
- `/debug` — toggle verbose debug output

### Planning & state
- `/plan` — open the plan editor
- `/todo` — manage the persistent todo list
- `/resume` — resume a saved session
- `/init` — scan the current project and write `AFK.md`
- `/changelog` — render `CHANGELOG.md` paginated

### Background subagent jobs
- `/bgsub` — list background subagent jobs with status (dispatched via `mode:'background'`)

### Skills (built-in)
- `/mint`, `/diagnose`, `/forge`, `/audit-fit`

See [`architecture.md`](architecture.md#skills--subagents) for what each does.

### Plugins / marketplaces
- `/skills` (alias `/builtin-skills`) — discover skills loaded from plugins & user scope
- `/agents` — list Task-tool subagents loaded by the SDK
- `/reload-plugins` — re-scan plugin and user directories after edits

Implementation: `src/cli/slash/index.ts` (`registerAll()`), individual command modules under `src/cli/slash/commands/`. Plugin-discovered skills (`~/.afk/plugins/<plugin>/skills/<skill>/SKILL.md` and `~/.afk/skills/<skill>/SKILL.md`) are registered via `src/cli/slash/builtin-skills.ts` and `src/cli/slash/plugin-skills.ts`.

## Models & providers

`agent-afk` speaks to two providers through a single abstraction (`src/agent/providers/`):

**Anthropic (direct)** — default. Selects from:
- **fable** — most capable (Claude Fable 5, Mythos-class) — hardest reasoning + long-horizon agentic work; 1M context
- **opus** — most capable Opus-tier, for complex tasks
- **sonnet** — balanced performance and speed (default)
- **haiku** — fastest, best for simple tasks

**Model slots** — four rebindable capability tiers (cheapest → most capable): `local`, `small`, `medium`, `large`. Use `AFK_MODEL_LOCAL` + `AFK_MODEL_LOCAL_BASE_URL` to point the `local` slot at Ollama, LM Studio, or any OpenAI-compatible shim. See [`docs/model-slots.md`](model-slots.md) for full configuration.

**OpenAI-compatible models** — set `AFK_MODEL=codex` (or `gpt-*`, `o1*`/`o3*`/`o4*`, or a HuggingFace-style `org/model` id), or pass `--model`. Implementation lives in `src/agent/providers/openai-compatible/`; `'openai-codex'` is a deprecated alias. Tune reasoning effort via `AFK_EFFORT=low|medium|high`.

## Plugins & marketplaces

The `/plugin` slash command (and marketplace-based install) is a Claude Code CLI feature, not an Agent SDK feature — see [anthropics/claude-code#15071](https://github.com/anthropics/claude-code/issues/15071). Inside an `agent-afk` session those commands don't exist, and marketplace installs run from Claude Code still land in `~/.claude/plugins/`. AFK ships its own `afk plugin` subcommand that keeps everything under `~/.afk/plugins/`:

```bash
# Install from a GitHub shorthand (expands to https://github.com/<owner>/<repo>.git)
afk plugin install anthropics/claude-plugins-official

# Install from an explicit git URL (https or ssh)
afk plugin install https://github.com/example/my-plugin.git

# Install from a local checkout (symlinks into ~/.afk/plugins/)
afk plugin install ~/projects/agent-afk/my-plugin

# Pin to a specific tag, branch, or SHA
afk plugin install owner/repo --ref v1.2.3

# List, update, and disable without deleting
afk plugin list
afk plugin update                                # all plugins
afk plugin update claude-plugins-official
afk plugin disable claude-plugins-official
afk plugin enable  claude-plugins-official
afk plugin remove  claude-plugins-official
```

By default `install` picks the highest semver git tag, falling back to the default branch when no tag parses as semver. State lives in `~/.afk/plugins/.index.json` — the scanner reads it at session startup and skips any entry with `enabled: false`.

**Advanced:** AFK still auto-discovers any plugin dropped into `~/.afk/plugins/<name>/` by hand (git clone, symlink, or copy). The directory just has to contain `.claude-plugin/plugin.json`; the scanner walks up to 5 levels deep so Claude Code's `cache/<marketplace>/<plugin>/<version>/` layout also works.

Plugin state (telemetry, ledger, briefs) writes to `~/.afk/agent-framework/` in the AFK runtime.

### Marketplaces

```bash
afk marketplace install <url>
afk marketplace list
afk marketplace remove <name>
```

State lives under `~/.afk/marketplaces/`.

## Telegram bot

Run Claude as a Telegram bot with full Agent SDK capabilities:

```bash
# 1. Get a bot token from @BotFather on Telegram
#    - Open Telegram, search @BotFather
#    - Send /newbot and follow instructions, copy the token
# 2. Run the setup wizard
afk telegram setup
# 3. Start the bot
afk telegram start
```

The setup wizard prompts for the token and the chat IDs allowed to interact with the bot (`AFK_TELEGRAM_ALLOWED_CHAT_IDS`).

**Bot commands:**
- `/start` — welcome
- `/clear` — clear conversation history
- `/model local|small|medium|large|opus|sonnet|haiku|fable` — switch model

Send any other message to chat. The bot runs in the default permission mode — tools execute without per-tool prompts, and file access outside the working directory stays contained.

**Background-service helpers:**

```bash
afk telegram start
afk telegram stop
afk telegram status
afk telegram restart
afk telegram logs
```

## Config home

`agent-afk` keeps its configuration and user-scope runtime state under `~/.afk/`, fully independent of Claude Code's `~/.claude/`:

```
~/.afk/
  config/              afk.env, afk.config.json
  state/               sessions/, todos/, transcripts/, daemon/  ($AFK_STATE_DIR overrides this tier)
  logs/
  cache/
  plugins/             user-installed Claude plugins (starts empty)
  agents/              user-defined subagents (SDK)
  commands/            user-defined slash commands (SDK)
  skills/              user-defined skills (SDK)
  settings.json        SDK settings (model, permissions, MCP, etc.)
  agent-framework/     forge telemetry + briefs
```

You can delete `~/.claude/` entirely and `agent-afk` still runs.

### `afk.config.json`

A minimal `afk.config.json` looks like:

```json
{
  "model": "sonnet",
  "maxTokens": 4096,
  "temperature": 1.0,
  "systemPrompt": "You are a helpful AI assistant."
}
```

See [`afk.config.json.example`](../afk.config.json.example) at the repo root for the full example. Resolution order: `<cwd>/afk.config.json` → `~/.afk/config/afk.config.json` → legacy `~/.afk.config.json`.
