# The AFK Contract

> Agent AFK already constrains execution. Now it needs to bear witness.

## Thesis

**AFK is unattended work with surviving evidence.**

The runtime may compress context. It must not erase evidence.
The runtime may delegate work. It must preserve who did what.
The runtime may block, abort, or cap execution. It must record why.
Surfaces are views over trace. Trace is the shared execution truth.

AFK does not mean "the agent kept running while I wasn't watching."
It means: the agent kept running, and when I came back, the runtime could reconstruct what happened from surviving evidence.

## Why this doc exists

The codebase has a strong enforcement philosophy. It blocks. It aborts. It caps. It fails closed. It has denylisted paths, hook gates, abort graphs, max depth, budgets, schema failure, partial-output rescue. The runtime has teeth.

The weak spot is witness:

- `SubagentTrace` exists but evaporates with the process.
- Hook blocks happen but do not become durable facts.
- `compact.ts` rewrites working memory; no record survives.
- Different surfaces show different realities.
- Session summaries literally say "session ended."

This doc is the contract the runtime owes the operator across that gap. Every subsequent PR justifies itself against it.

## The contract

### What is impossible by construction

The runtime refuses these regardless of what the model requests:

- Writes to denylisted paths (`~/.ssh`, `~/.aws`, `~/.gnupg`, `~/.config/gcloud`, `/etc`, `/System`, `/private/etc`, `/usr/local/etc`); denylist is non-removable via env.
- `write_file` / `edit_file` in plan mode.
- Recursive `compose` DAGs (`compose` excluded from `CHILD_ALLOWED_TOOLS`).
- Subagent nesting deeper than `DEFAULT_MAX_NESTING_DEPTH` (currently 3).
- Ignoring an abort signal mid-dispatch (abort is checked before and between every hook await).
- Silent hook failure (handler exceptions become `HookBlockedError`, fail-safe).
- Exceeding `maxTurns` on a session.
- Exceeding `maxBudgetUsd` when the provider reports `totalCostUsd`.

### What is merely best-effort

The runtime labels these honestly as conventions, not walls:

- Plan-mode `bash` denial — substring-matched, does not catch interpreter-mediated writes (`python -c`, heredocs, `dd`, etc.). The code itself states this is a "best-effort honesty primitive, not a security boundary."
- Budget enforcement on providers that omit `totalCostUsd` (silently inert).
- Untraced tool use when the dispatcher is constructed without a hook registry.
- Child subagent `canUseTool` — a fork can be given **broader** permissions than its parent. There is no enforced narrowing.

These are documented gaps. They are honest because they are named.

### What must be traced

A session is legitimate AFK work only if these events were recorded:

- Every tool call (name, input size, result size, error flag, truncation flag).
- Every hook decision (event, decision kind, reason).
- Every subagent lifecycle transition (start, succeeded, failed, cancelled).
- Every budget event (limit, observed value, action taken).
- Every abort (origin, cascade target).
- Every compaction (pre-compaction message slice, replacement summary).
- Every closure (reason: `model_end_turn` / `iteration_cap` / `abort` / `timeout` / `budget_exceeded`).
- Every structured claim emitted by an agent, subagent, skill, or verifier (assertion, evidence, confidence, dissent).
- The terminal seal (`session_sealed`, written and fsync'd before exit).

### What must survive compaction

Working memory may compress. Witness memory must not.

`compact.ts` may replace the in-flight conversation history with a model-generated summary for context efficiency. But the trace event for that compaction must preserve the **pre-compaction message slice full-fidelity**. The summary is itself a lossy artifact generated by a model call; if the operator wants to audit what was forgotten, only the original survives.

The single invariant: **working memory may compress; witness memory must not.**

### How surfaces view trace

Surfaces are not independent renderers. They are views over trace. Each surface declares which event kinds it renders at which density:

- **REPL** — dense live tail (tool lanes, thinking, hook blocks, closure).
- **Telegram** — sparse high-signal events (subagent start/end, hook blocks, closure, abort).
- **Daemon** — post-hoc rollup derived from trace.
- **`afk show <id>`** — full replay.
- **`afk tail <id>`** — live tail of an in-flight session.

The trace is the source. Surfaces choose filter and density. No surface invents reality.

### What closure means

A session reaches closure for one of these reasons, and the reason is recorded:

- `model_end_turn` — provider emitted a non-`tool_use` stop reason.
- `iteration_cap` — inner tool-use loop hit its hard ceiling.
- `abort` — external signal cascaded through the abort graph.
- `timeout` — per-turn or per-session deadline elapsed.
- `budget_exceeded` — USD ceiling tripped mid-stream.
- `hook_blocked` — a hook blocked a load-bearing dispatch and the run could not recover.

Closure quality (was the work actually finished, were tests passing, was the question answered) is delegated to the model and to skill-level contracts. The runtime only owns the closure *reason*, not the closure *quality*.

### When a child can or cannot exceed parent authority

A fork inherits the parent's `canUseTool` by default via permission bubbling. A fork **may** be given an explicit `canUseTool` that widens authority beyond the parent's; this is currently possible by construction and is **named as a gap**, not endorsed.

The contract direction (not yet enforced): a child may narrow but not widen its parent's authority. Widening should require an explicit, traced grant.

## The witness layer (v1 scope)

The first PR that satisfies this contract introduces an append-only trace layer.

### Storage

One file per session: `~/.afk/state/sessions/<session-id>/trace.jsonl`

Append-only NDJSON. Each line:

```json
{ "ts": "2026-05-17T12:34:56.789Z", "seq": 42, "kind": "tool_call", "payload": { ... } }
```

`seq` is a monotonic counter so out-of-order reads can sort. `ts` is ISO-8601 with millisecond precision.

### Event kinds

```
tool_call
hook_decision
subagent_lifecycle
budget
abort
compaction
closure
claim
session_sealed
```

Payload schemas are defined in `src/agent/trace/events.ts` (forthcoming). Each kind's payload is a small, committed taxonomy — the schema is *part of the contract*, not free-form metadata.

### Session states

A trace file is in exactly one of three states:

- **live** — no `session_sealed` record; file is being appended to.
- **sealed-clean** — a `session_sealed` record terminates the file with `status: succeeded | failed | cancelled` plus final cost and turn count.
- **sealed-crashed** — no `session_sealed` record exists after the owning process exits or the runtime marks the session abandoned. This is itself a piece of evidence: the runtime tried to record and did not seal.

### Out of scope for v1

These belong to a future v2 and are explicitly **not** promised by v1:

- Cryptographic integrity (hash chains, signed timestamps).
- Cross-session indexing (FTS, SQLite mirror).
- Trace garbage collection or retention policy.
- Replay-driven re-execution.
- A claim arbitration protocol (claims are *recorded* in v1, not adjudicated).

V1 is operator-narrative. "Here's what happened while you were AFK." Audit-grade integrity is a layer we can build on top of the same event log without breaking the narrative reader.

## The line for the wall

> Agent AFK already constrains execution. Now it needs to bear witness.

Engineering corollary: **trace is the first feature that makes every other philosophical question concrete.** Until the trace is real, this doc is aspirational. Once the trace is real, this doc is enforceable.
